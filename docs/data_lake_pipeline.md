# MedFlow Data Lake And Cleanup Pipeline

## Current Direction

The project is aligned with the instructor's plan:

1. Collect raw source data first.
2. Clean and normalize into reviewable candidate files.
3. Only then load validated data into PostgreSQL clinical tables.

The repository now keeps raw source files under `knowledge_base/sources/` and cleaned pipeline outputs under `knowledge_base/sources/clean/`.

## Raw Data Lake

### Tunisian PCT Human Medicines

Source:

- `http://www.phct.com.tn/index.php/catalogue/medicament-humain/form/25/`

Files:

- `knowledge_base/sources/pct_human_medicines_raw.csv`
- `knowledge_base/sources/pct_human_medicines_reference_options.csv`
- `knowledge_base/sources/pct_human_medicines_recent_rich_rows.csv`

Purpose:

- Capture Tunisian market medicine labels as raw evidence.
- Preserve brand-like labels, strengths, and source URL.
- Do not load directly into clinical tables without normalization.

### RxNorm

Files:

- `knowledge_base/sources/rxnorm_mapping.json`
- `knowledge_base/sources/rxnorm_mapping.csv`

Purpose:

- Map the 30 priority molecules to RxNorm CUIs and synonyms.
- Support therapeutic duplication detection.

### ChEMBL

File:

- `knowledge_base/sources/chembl_drug_data.csv`

Purpose:

- Store CYP relationships and molecular identifiers for the priority molecules.
- Support CYP-mediated risk detection.

### OpenFDA

File:

- `knowledge_base/sources/openfda_drug_data.csv`

Purpose:

- Cross-check warnings, contraindications, and interaction language.
- OpenFDA is secondary evidence, not the main severity authority.

### ANSM Interactions

File:

- `knowledge_base/sources/ansm_interactions_all.csv`

Purpose:

- Store Week 1 clinically important interaction pairs for the 30 priority molecules.
- This is currently a curated/generated starter file, not a full parsed ANSM thesaurus.

DrugBank is intentionally excluded for now because the team could not obtain access for Tunisia.

## Allergy Layer

The allergy seed layer supports the Week 1 allergy trap:

- Patient has documented penicillin anaphylaxis.
- Incoming prescription is amoxicillin.
- Expected result is an allergy conflict alert.

The cleaned allergy files currently encode:

- Allergy group definitions.
- Molecule-to-allergy-group memberships.
- Cross-reactivity rules, such as `Penicillins <-> Cephalosporins` and `Salicylates <-> NSAIDs`.

## Removed Unsafe Path

The old `pct_all_medicines.csv` / `load_all_pct_drugs.py` path was removed because it created molecules from brand names when the INN was missing. That would pollute the clinical database with fake molecules such as brand labels. The project should use the cleaned candidate files instead.

## Database Loading

Use:

```powershell
python knowledge_base/loaders/run_all_loaders.py
```

The loader sequence expects PostgreSQL to be running and source CSVs to exist. The `load_pct_brands.py` loader now reads `sources/clean/tunisian_brand_mapping_clean.csv`.

## Validation

Use:

```powershell
python evaluation/manual_e2e_test.py
```

This verifies the first manual end-to-end trap:

- Trap patient 1 is actively taking warfarin.
- Incoming prescription is aspirin.
- The database returns the expected interaction severity, clinical effect, and management recommendation.
